# **Background: everything you need to know**

To solve the problem of determining whether a given integerÂ `n`Â is a power of two, we need to consider the most efficient approach possible. This means avoiding loops or recursion and leveraging bitwise operations, which are both fast and low-overhead.

### **Solution Strategy**

A numberÂ `n`Â is a power of two if it has exactly one bit set in its binary representation. For example:

- `1`Â (binaryÂ `0001`) isÂ `2^0`
- `2`Â (binaryÂ `0010`) isÂ `2^1`
- `4`Â (binaryÂ `0100`) isÂ `2^2`

In binary, a power of two has exactly oneÂ `1`Â bit and the rest areÂ `0`s. This characteristic allows us to use a simple bitwise operation to check ifÂ `n`Â is a power of two.

### **Bitwise Trick Explanation**

For any power of two,Â `n`, the expressionÂ `n & (n - 1)`Â will beÂ `0`. This is because:

- `n`Â has exactly one bit set toÂ `1`.
- `n - 1`Â flips all the bits after the rightmostÂ `1`Â inÂ `n`, including theÂ `1`Â itself.

Therefore:

- `n & (n - 1)`Â clears the onlyÂ `1`Â bit inÂ `n`, resulting inÂ `0`Â ifÂ `n`Â is a power of two.

# **Java Solution (Runtime 0ms Beats 100.00%)**

```java
class Solution {
    public boolean isPowerOfTwo(int n) {
        // A number n is a power of two if it is greater than 0 and (n & (n - 1)) is 0
        return n > 0 && (n & (n - 1)) == 0;
    }
}
```

### **Explanation of the Code**

1. **ConditionÂ `n > 0`:**
   - We first check ifÂ `n`Â is greater thanÂ `0`Â because a non-positive number (0 or negative) cannot be a power of two.
2. **Bitwise OperationÂ `(n & (n - 1))`:**
   - This checks ifÂ `n`Â has exactly one bit set in its binary representation. IfÂ `n`Â is a power of two,Â `(n & (n - 1))`Â will beÂ `0`, confirming thatÂ `n`Â is indeed a power of two.
3. **Return Value:**
